Modul 03 · Tier 1
EntityManager & Lifecycle
Der zentrale Einstiegspunkt zu JPA — Zustände, Transaktionen und CRUD.
💡 Warum wichtig? Jede JPA-Operation läuft über den EntityManager. Das Lifecycle-Modell (NEW → MANAGED → DETACHED → REMOVED) ist Kern der mündlichen Matura-Prüfung.
// EntityManager — Zustandsübergänge
NEW ──→── persist() → in Context aufnehmen MANAGED ──→── remove() → zum Löschen markiert MANAGED ──→── detach() → aus Context entfernt DETACHED ──→── merge() → neue managed Kopie MANAGED ──→── flush() → SQL sofort ausführen

📋 JPA Cheat Sheet
Entity-Grundlagen
@EntityKlasse als JPA-Entity markieren
@Table(name=)Tabellennamen überschreiben
@IdPrimärschlüssel-Feld
@GeneratedValuestrategy: IDENTITY, SEQUENCE, AUTO
@VersionOptimistic Locking
@EmbeddedEingebettetes Objekt verwenden
@EmbeddableKlasse als einbettbar markieren
Column Mapping
@Column(...)name, length, nullable, updatable, unique, scale, precision
@TransientFeld nicht persistieren
@LobCLOB (String) oder BLOB (byte[])
@EnumeratedEnumType.STRING oder ORDINAL
@TemporalNur für java.util.Date nötig
EntityManager
em.persist(e)NEW → MANAGED
em.find(C,id)Laden per PK
em.merge(e)DETACHED → neue MANAGED Kopie
em.remove(e)MANAGED → REMOVED
em.flush()SQL sofort ausführen
em.clear()Context leeren
Relationships
@OneToManymappedBy, cascade, orphanRemoval
@ManyToOnefetch = LAZY (Standard empfohlen)
@ManyToMany@JoinTable für Zwischentabelle
@JoinColumnFK-Spaltenname definieren
CascadeTypeALL, PERSIST, MERGE, REMOVE, REFRESH
Vererbung
SINGLE_TABLEAlle in eine Tabelle — schnell, NULLs
JOINEDNormalisiert — JOIN nötig
TABLE_PER_CLASSEigene Tabelle je konkreter Klasse
@MappedSuperclassBasis ohne eigene Tabelle
@DiscriminatorValueTyp-Wert in Diskriminator-Spalte
JPQL & Panache
createQuery(jpql)JPQL-Query (Klassenname, nicht Tabelle!)
createNamedQueryVordefinierte @NamedQuery aufrufen
setParameter(k,v)Parameter binden
@TransactionalCDI/Quarkus — automatische Tx
PanacheEntityActive Record Pattern
PanacheRepositoryRepository Pattern
